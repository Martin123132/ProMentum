# ProMentum v0.2.0 Release Review

Status: local review draft. Do not tag, publish, or create a GitHub Release until Martin approves the inspected app and final ZIP.

## Headline

ProMentum v0.2.0 adds Project Momentum: a guided workflow that turns a generated spark into a small local project.

## Workflow

```text
Capture -> Generate -> Choose -> Shape -> Do Next
```

## What Changed

- Added a new **Momentum** page between Result and Library.
- Added local `projects.json` storage under `PROMENTUM_HOME`.
- Added **Save as Project** from the Result page.
- Added project editing for title, workflow stage, traffic-light readiness, notes, and small actions.
- Added project action checkboxes so users can return later and see what moved.
- Added **Copy Project Brief** and **Export Project Card**.
- Added `GET /api/projects`, `POST /api/projects`, `DELETE /api/projects`, and `POST /api/projects/export`.
- Updated doctor output with project count and projects path.
- Updated tests, release ZIP verification, screenshots, README, and UX smoke checklist.

## Local-First Promise

- No API keys.
- No cloud calls.
- No accounts.
- No telemetry.
- No npm or build step.
- Runtime data stays local and can be directed with `PROMENTUM_HOME`.

## Review Checklist

1. Open the local review app.
2. Confirm the status pill says `ProMentum v0.2.0 - Ready`.
3. Generate a result.
4. Click **Save as Project**.
5. Confirm Momentum opens and shows the saved project.
6. Add one action.
7. Change the traffic light.
8. Save the project.
9. Copy the project brief.
10. Export the Project Card.
11. Reopen the app and confirm the project is still there.
12. Check desktop and mobile widths for overflow.
13. Stop the app server before handing the machine back.
