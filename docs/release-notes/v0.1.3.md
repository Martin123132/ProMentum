# ProMentum v0.1.3 Release Review

Status: local review draft. Do not tag, publish, or create a GitHub Release until Martin approves the inspected app and final ZIP.

## Headline

ProMentum v0.1.3 is a first-run confidence and shareability patch. It keeps the same local-first promise while making the Generate and Result pages clearer for someone opening the app for the first time.

## What Changed

- Added a first-run guidance card on the Generate page with a one-click friendly defaults action.
- Added a standalone Share Card HTML export for the best hook, next move, and repeatable recipe.
- Added copyable share-summary text for quick tester feedback or social snippets.
- Updated README screenshots for the Start, Result, and Share Card flows.
- Expanded tests around share-card rendering, export behavior, UI smoke coverage, and release ZIP contents.
- Bumped the app and package version to `0.1.3`.

## Local-First Promise

- No API keys.
- No cloud calls.
- No accounts.
- No telemetry.
- No npm or build step.
- Runtime data stays local and can be directed with `PROMENTUM_HOME`.

## Review Checklist

1. Open the local review app.
2. Confirm the status pill says `ProMentum v0.1.3 - Ready`.
3. Use **Use First-Run Defaults** on the Generate page.
4. Generate a result.
5. Copy the share summary.
6. Export the Share Card.
7. Save a favourite.
8. Confirm the exports folder is understandable.
9. Check desktop and mobile widths for overflow.
10. Stop the app server before handing the machine back.
